## Getting started

To get started, ensure you have the [Prisma Client](https://www.prisma.io/docs/concepts/components/prisma-client) updated to version `5.7.0` or later. This is necessary to access the Platform through the Prisma CLI.

<Admonition>

ðŸ’¡ When using commands, always start with `prisma platform` and include the `--early-access` flag to enable the use of the Prisma Data Platform whilst still in early access.

</Admonition>

## Authentication

### <inlinecode>platform</inlinecode>

#### <inlinecode>auth login</inlinecode>

Logs into your Prisma Data Platform account or creates a new one.

```bash
npx prisma platform auth login --early-access
```

#### <inlinecode>auth logout</inlinecode>

Logs out of your Prisma Data Platform account.

```bash
npx prisma platform auth logout --early-access
```

#### <inlinecode>auth show</inlinecode>

Displays information about the currently authenticated user.

```bash
npx prisma platform auth show --early-access
```

### Workspace Management

### <inlinecode>platform</inlinecode>

#### <inlinecode>workspace show</inlinecode>

Lists all workspaces available to your account.

```bash
npx prisma platform workspace show --early-access
```

### Project Management

### <inlinecode>platform</inlinecode>

#### <inlinecode>project show</inlinecode>

Lists all projects within the specified workspace.

```bash
npx prisma platform project show \
--workspace INSERT_WORKSPACE_ID \
--early-access
```

##### Arguments

| Argument           | Type     | Required | Description                                                                                                                 |
| ------------------ | -------- | -------- | --------------------------------------------------------------------------------------------------------------------------- |
| `--workspace` `-w` | `string` | yes      | The workspace id.<br/><br/> **Hint:** You can view your workspace ids with the [`workspace show`](#workspace-show) command. |

#### <inlinecode>project create</inlinecode>

Creates a new project within the specified workspace.

```bash
npx prisma platform project create \
--workspace INSERT_WORKSPACE_ID \
--name "INSERT_PROJECT_NAME" \
--early-access
```

##### <span>Arguments</span>

| Argument           | Type     | Required | Description                                                                                                                   |
| ------------------ | -------- | -------- | ----------------------------------------------------------------------------------------------------------------------------- |
| `--workspace` `-w` | `string` | yes      | The workspace `id`.<br/><br/> **Hint:** You can view your workspace ids with the [`workspace show`](#workspace-show) command. |
| `--name` `-n`      | `string` | no       | The display name for the project.<br/><br/> If omitted, a default project name will be generated for you.                     |

#### <inlinecode>project delete</inlinecode>

Deletes the specified project.

```bash
npx prisma platform project delete \
--workspace INSERT_WORKSPACE_ID \
--project INSERT_PROJECT_ID \
--early-access
```

##### <span>Arguments</span>

| Argument           | Type     | Required | Description                                                                                                                   |
| ------------------ | -------- | -------- | ----------------------------------------------------------------------------------------------------------------------------- |
| `--workspace` `-w` | `string` | yes      | The workspace `id`.<br/><br/> **Hint:** You can view your workspace ids with the [`workspace show`](#workspace-show) command. |
| `--project` `-p`   | `string` | yes      | The project `id`.<br/><br/>\*_Hint_ \*: You can view your project ids with the [`project show`](#project-show) command.       |

### API Key Management

Lists all API keys for the specified project.

```bash
npx prisma platform apikey show \
--workspace INSERT_WORKSPACE_ID \
--project INSERT_PROJECT_ID \
--early-access
```

##### Arguments

| Argument         | Type     | Required | Description                                                                                                                   |
| ---------------- | -------- | -------- | ----------------------------------------------------------------------------------------------------------------------------- |
| `--workspace -w` | `string` | yes      | The workspace `id`.<br/><br/> **Hint:** You can view your workspace ids with the [`workspace show`](#workspace-show) command. |
| `--project` `-p` | `string` | yes      | The project `id`.<br/><br/>\*_Hint_ \*: You can view your project ids with the [`project show`](#project-show) command.       |

#### <inlinecode>apikey create</inlinecode>

Creates a new API key for the specified project.

```bash
npx prisma platform apikey create \
--workspace INSERT_WORKSPACE_ID \
--project INSERT_PROJECT_ID \
--name "INSERT_API_KEY_NAME" \
--early-access
```

##### <span>Arguments</span>

| Argument           | Type     | Required | Description                                                                                                                   |
| ------------------ | -------- | -------- | ----------------------------------------------------------------------------------------------------------------------------- |
| `--workspace` `-w` | `string` | yes      | The workspace `id`.<br/><br/> **Hint:** You can view your workspace ids with the [`workspace show`](#workspace-show) command. |
| `--project` `-p`   | `string` | yes      | The project `id`.<br/><br/>**Hint**: You can view your project ids with the [`project show`](#project-show) command.          |
| `--name` `-n`      | `string` | no       | The display name for the API key.<br/><br/> If omitted, a default API key name will be generated for you.                     |

##### <span>Arguments</span>

| ---------------- | -------- | -------- | ----------------------------------------------------------------------------------------------------------------------------- |
| `--workspace -w` | `string` | yes      | The workspace `id`.<br/><br/> **Hint:** You can view your workspace ids with the [`workspace show`](#workspace-show) command. |
| `--project` `-p` | `string` | yes      | The project `id`.<br/><br/>**Hint**: You can view your project ids with the [`project show`](#project-show) command.          |
| `--apikey`       | `string` | yes      | The API key `id`.<br/><br/>**Hint**: You can view your API key ids with the [`apikey show`](#apikey-show) command.            |

### Prisma Accelerate

### <inlinecode>platform</inlinecode>

#### <inlinecode>accelerate enable</inlinecode>

Enables Prisma Accelerate for the specified project.

```bash
npx prisma platform accelerate enable \
--workspace INSERT_WORKSPACE_ID \
--project INSERT_PROJECT_ID \
--url "postgresql://username:password@host:port/database"
--region INSERT_CONNECTION_POOL_REGION
--apikey true
--early-access
```

##### <span>Arguments</span>

| Argument         | Type      | Required | Description                                                                                                                                                                                                                                                        |
| ---------------- | --------- | -------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| `--workspace -w` | `string`  | yes      | The workspace `id`.<br/><br/> **Hint:** You can view your workspace ids with the [`workspace show`](#workspace-show) command.                                                                                                                                      |
| `--project` `-p` | `string`  | yes      | The project `id`.<br/><br/>**Hint**: You can view your project ids with the [`project show`](#project-show) command.                                                                                                                                               |
| `--url`          | `string`  | yes      | Your database connection string.                                                                                                                                                                                                                                   |
| `--region`       | `string`  | no       | The region for Accelerateâ€™s managed connection pool.<br/><br/>View a list of available regions [here](/accelerate/faq#what-regions-is-accelerates-connection-pool-available-in).<br/><br/>**Hint**: Select the region _nearest_ your database for optimal latency. |
| `--apikey`       | `boolean` | no       | If yes, a new API key will be generated for the associated project.                                                                                                                                                                                                |

#### <inlinecode>accelerate disable</inlinecode>

Disables Prisma Accelerate for the specified project.

```bash
npx prisma platform accelerate disable \
--workspace INSERT_WORKSPACE_ID \
--project INSERT_PROJECT_ID \
--early-access
```

##### <span>Arguments</span>

| Argument         | Type     | Required | Description                                                                                                                   |
| ---------------- | -------- | -------- | ----------------------------------------------------------------------------------------------------------------------------- |
| `--workspace -w` | `string` | yes      | The workspace `id`.<br/><br/> **Hint:** You can view your workspace ids with the [`workspace show`](#workspace-show) command. |
| `--project` `-p` | `string` | yes      | The project `id`.<br/><br/>**Hint**: You can view your project ids with the [`project show`](#project-show) command.          |
